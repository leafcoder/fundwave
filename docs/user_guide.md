## 快速开始

### 系统要求

- **操作系统**: Windows 10+, macOS 10.14+, Ubuntu 18.04+
- **Python版本**: 3.8+
- **内存**: 最低 2GB RAM（推荐 4GB+）
- **磁盘空间**: 100MB 可用空间
- **网络**: 需要互联网连接（用于获取基金数据）

### 安装步骤

#### 方式一：直接运行（推荐）

```bash
# 1. 克隆或下载项目代码
cd fundwave

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动程序
python main.py
```

#### 方式二：使用启动脚本（Linux/macOS）

```bash
# 赋予执行权限
chmod +x start.sh

# 运行程序
./start.sh
```

**注意**: 如果遇到Qt平台插件错误，请务必使用 `start.sh` 脚本启动。

### 首次启动

程序首次启动时会自动：

1. ✅ 创建本地数据库 `fund_monitor.db`
2. ✅ 加载默认监控基金列表（18只热门基金）
3. ✅ 自动获取最新估值数据
4. ✅ 显示主界面窗口

## 操作指南

### 1️⃣ 添加基金监控

#### 方法一：通过添加按钮

1. 点击工具栏的 **【添加基金】** 按钮
2. 在弹出的对话框中输入6位基金代码
3. 点击 **【添加】** 按钮
4. 确认后自动刷新数据

#### 方法二：通过搜索功能

1. 点击顶部的 **【基金搜索】** 标签页
2. 在搜索框中输入基金代码或名称（如"华夏"）
3. 从结果列表中点击选中目标基金
4. 点击 **【添加到监控】** 按钮
5. 自动切换回数据表页面

#### 方法三：右键菜单

1. 在搜索结果表格中 **右键点击** 目标基金
2. 选择 **【添加基金监控】**

### 2️⃣ 设置持仓信息

#### 设置持仓成本价和持有份额

1. 在基金数据表中 **右键点击** 目标基金行
2. 选择 **【设置持仓成本价】** 或 **【设置持有份额】**
3. 在弹出对话框中输入数值
4. 点击确定，系统会自动重新计算盈亏

**💡 小提示**: 
- 持仓成本价：您购买时的平均价格
- 持有份额：您持有的基金总份数（可在基金App中查看）
- 这两个值是计算盈亏的基础

#### 记录分红

当您收到基金分红时：

1. 右键点击该基金 → 选择 **【记录分红】**
2. 输入分红金额（单位：元）
3. 系统会将分红计入累计盈亏，确保收益计算准确

**示例**:
- 持有1000份，成本价1.5元 → 持仓成本 = ¥1500
- 当前净值1.8元 → 持有金额 = ¥1800
- 收到分红¥50
- **真实累计盈亏** = 1800 - 1500 + 50 = **¥350** （而非¥300）

### 3️⃣ 隐藏敏感数据（隐私保护）

如果您在公共场合使用，可以隐藏金额数据：

#### 隐藏统计卡片数据

在顶部统计栏中：

- **今日盈亏**、**持仓成本**、**持有金额** 旁边都有 👁 图标
- 点击 👁 → 数据变为 `¥****`，图标变为 🔒
- 再次点击 🔒 → 恢复显示

✨ **新特性**: 隐藏状态会自动保存，下次启动程序仍然保持！

#### 隐藏累计盈亏

在标题栏右侧：

- 点击累计盈亏旁边的 👁 图标
- 同样支持状态持久化

### 4️⃣ 自定义刷新设置

#### 调整刷新间隔

1. 在 **刷新间隔** 输入框中修改秒数（建议：30-300秒）
2. 点击 **【设置】** 按钮
3. 生效后定时器将按照新间隔刷新

**推荐设置**:
- 交易时间（9:30-15:00）：30-60秒
- 非交易时间：300秒以上（节省资源）

#### 启动/停止自动刷新

- 点击 **【停止刷新】** → 手动模式（需点击【手动刷新】）
- 点击 **【启动刷新】** → 恢复自动刷新

### 5️⃣ 配置通知服务

#### 弹窗通知（默认开启）

当满足以下条件时弹出系统通知：

- 单只基金涨幅 ≥ 3%（可自定义阈值）
- 单只基金跌幅 ≤ -3%
- 累计盈利 ≥ ¥1000
- 累计亏损 ≤ -¥1000

#### 钉钉机器人通知

1. 点击 **【通知设置】** 按钮
2. 勾选 **【启用钉钉通知】**
3. 输入 Webhook 地址（从钉钉群机器人获取）
4. 设置预警阈值
5. 点击 **【保存】**

**获取钉钉Webhook**:
1. 打开钉钉群 → 群设置 → 智能群助手
2. 添加机器人 → 自定义（通过Webhook接入）
3. 复制Webhook地址

### 6️⃣ 使用系统托盘

#### 最小化到托盘

- 点击窗口右上角 **×** 关闭按钮
- 程序不会退出，而是最小化到系统托盘

#### 从托盘恢复

- **双击** 托盘图标 → 显示主窗口
- **右键** 托盘图标:
  - **显示主界面**
  - **退出程序**（完全关闭）

### 7️⃣ 表格操作技巧

#### 排序

- 点击任意列标题 → 按该列排序（升序/降序切换）
- 数字列（金额、百分比）会按数值大小正确排序

#### 调整列宽

- 拖动列标题之间的分隔线
- 所有列都支持调整宽度

#### 隐藏/显示列

1. 在表格空白处 **右键点击**
2. 选择 **【显示/隐藏列】**
3. 勾选/取消勾选需要显示的列

#### 查看历史图表

1. 右键点击某只基金
2. 选择 **【查看历史净值走势】**（如果启用）

## 常见问题

### Q1: 程序启动报错 "Could not find the Qt platform plugin"

**解决方案**: 
```bash
# 使用提供的启动脚本
./start.sh
```

或者手动设置环境变量：
```bash
export QT_QPA_PLATFORM="xcb"
python main.py
```

### Q2: 数据不更新/显示旧数据

**可能原因及解决**:

1. **网络问题**: 检查网络连接
2. **非交易时间**: 周末和节假日估值不更新（正常现象）
3. **API限制**: 天天基金接口可能有频率限制，稍后再试

**手动刷新**: 点击工具栏的 **【手动刷新】** 按钮

### Q3: 如何查看我的总资产？

查看顶部统计栏：

- **持仓成本**: 总投入本金
- **持有金额**: 当前市值
- **今日盈亏**: 今日浮动盈亏
- **累计盈亏**: 总盈亏（含分红）

### Q4: 盈亏计算不准确？

请确认已正确设置：

1. ✅ **持仓成本价**（不是当前净值！）
2. ✅ **持有份额**（精确到小数点后两位）
3. ✅ **分红记录**（如有分红必须记录）

**验证方法**:
```
持仓成本 = 成本价 × 份额
持有金额 = 估算值 × 份额
累计盈亏 = 持有金额 - 持仓成本 + 分红总额
```

### Q5: 如何备份我的数据？

所有数据存储在本地数据库文件 `fund_monitor.db`。

**备份方法**:
```bash
cp fund_monitor.db fund_monitor_backup_$(date +%Y%m%d).db
```

**重要数据包括**:
- 监控基金列表
- 持仓信息（成本价、份额）
- 分红记录
- 通知设置
- UI偏好设置（眼睛图标状态等）

### Q6: 升级到新版本如何保留数据？

数据库文件 `fund_monitor.db` 会自动保留，升级只需：

```bash
# 备份旧数据（可选但推荐）
cp fundwave/fund_monitor.db ./backup.db

# 更新代码
git pull  # 或下载新版本

# 安装新依赖（如果有变化）
pip install -r requirements.txt

# 启动程序（数据自动迁移）
python main.py
```

### Q7: 程序占用多少资源？

**典型资源占用**（18只基金）:

- 内存：~150MB
- CPU：空闲时 < 1%，刷新时 ~5%（瞬间）
- 网络：每次刷新约 ~50KB（18只基金）
- 磁盘：~10MB（数据库文件）

**优化建议**:
- 减少监控基金数量
- 增大刷新间隔
- 非交易时间停止自动刷新
